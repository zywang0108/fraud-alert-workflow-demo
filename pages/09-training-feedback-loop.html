步骤 09

SFT / RLVR / RLHF 反馈闭环

训练闭环分别优化 Agent Loop 的取证路由、Guardrails/Evidence/State 的可验证正确性，以及调查员认可的报告判断。

SFT → Agent Loop 路由 行为模仿
01
下一步动作策略 从 Context 学下一步：查库、扩图、取 Policy/Profile、跑 Python、停止或转人工。
02
参数填充 学习 account_id、window、feature_id、query_template_id、threshold、depth、row_limit。
03
停止 / 缺证信号 学习何时继续补证、何时输出缺证声明、何时进入 ready_for_report。
RLVR → Guardrails / Evidence / State 可验证奖励
01
Guardrail 通过 奖励合法 schema、权限范围、query_template、预算和人工边界。
02
Evidence 正确性 奖励 Query 可执行、计算可复算、Evidence card 可追溯。
03
State 转移 奖励正确更新 Claim 状态、避免重复 Query、保留回放记录。
RLHF → 人审判断 专家偏好
01
报告可用性 学习更清楚、更短、更适合调查员复核的报告结构。
02
风险措辞 学习“高度可疑 / 需进一步核验 / 证据不足”的措辞边界。
03
升级偏好 学习何时建议升级、何时保留人工复核，不能替代最终处置。
SFT 优化路由节点：Tool 选择、arguments、停止/缺证。
RLVR 优化可验证行为：Guardrail 通过、Evidence 正确性、State 转移。
RLHF 优化人审体验：报告可读性、风险措辞、升级偏好。
