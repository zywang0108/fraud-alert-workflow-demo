步骤 04

Context Layer / Case Memory

把告警、工具返回和 Case State 压成每轮最小 Context，避免凭记忆或全量明细判断。

Context 包 输入与控制
01
告警包 alert_id、account_id、risk_type、features、时间窗口。
02
可用 Tool 只注入本案允许的 Tool schema。
03
历史观察结果 摘要、Evidence 引用、query_hash、tool_call_id、缺口。
04
Case State Claim 状态与下一步证据缺口。
防止重复查证 已核验、已查询、空结果都进入 Context。
控制 Agent 可见数据范围 只注入告警摘要、聚合结果、必要样例行和 query reference；完整客户明细保留在受限 Database / Case Store 中。
减少上下文漂移 每轮从 Case Store 重建，不靠长对话记忆。
保证报告可追溯 报告只能引用 Context 中已有证据或缺证标记。
