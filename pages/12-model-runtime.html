步骤 12

模型运行方式

模型可替换；工具、状态、证据和审计协议保持不变。

运行时抽象 API / GPU / CPU
API 模式 复杂报告、解释和 Policy 推理。
自有 GPU 集群模式 Qwen / DeepSeek 通过兼容接口接入运行框架。
CPU 机器模式 轻量模型承担低并发、批处理和夜间回放。
统一审计边界 Prompt、Model、tool_call、hashes 全部进入审计日志。
01
runtime_policy 选择后端 按任务、敏感度、成本、延迟和合规选择后端。
02
工具与状态不随模型变化 工具、Case Store、Evidence card、Guardrails 不变。
03
复杂推理可降级 模型不可用时降级为模板、规则、只读查询或人工。
04
架构价值 业务能力不绑定模型供应商或硬件形态。
